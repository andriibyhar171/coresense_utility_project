Курсовий проєкт з дисципліни "Системне програмне забезпечення".

Згідно з методичними вказівками до виконання курсового проєкту:

Тема №3: Моніторинг використання ресурсів ПК
Завдання №41: Розробка утиліти для моніторингу системи.
Розробка програми, що дозволяє моніторити систему на комп'ютері з операційною системою Windows та отримувати інформацію про поточний стан системи.

|Функціональні можливості (Features)|:

1. Моніторинг процесора (CPU): Відображення загального навантаження на ЦП та навантаження по окремих ядрах у відсотках.
2. Аналіз оперативної пам'яті (RAM): Виведення інформації про загальний обсяг, доступну пам'ять, файл підкачки (Page File) та віртуальну пам'ять.
3. Керування процесами: Перегляд списку активних процесів із відображенням їх PID (Process ID), використаної пам'яті та пріоритету.
4. Детальна інформація про потоки: Можливість перегляду кількості потоків (threads) для обраного процесу.
5. Завершення процесів: Функція примусового завершення (Kill Process) завислих або небажаних програм через API виклики.
6. Моніторинг файлової системи: Відображення списку логічних дисків, їх типу, загального обсягу та вільного місця.
7. Мережева статистика: Відображення активності мережевих інтерфейсів (швидкість відправки та отримання пакетів).
8. Інформація про систему: Виведення даних про версію ОС, час роботи системи (System Uptime) та ім'я комп'ютера.
9. Логування показників: Можливість запису статистики використання ресурсів у текстовий файл для подальшого аналізу.
10. Сповіщення про навантаження: Система візуальних сповіщень (Alerts), коли використання CPU або RAM перевищує встановлений користувачем поріг (наприклад, >90%).
